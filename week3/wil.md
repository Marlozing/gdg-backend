# 3주차 강의 내용
데이터베이스에 접근하기 위해서는 SQL을 사용  
Java의 경우 JDBC를 이용해 데이터베이스에 접근  
그러나 SQL을 직접 이용할 경우 반복적으로 SQL을 사용해야 하고 개발이 SQL에 의존적이게 됨  
따라서 자바 진영의 ORM 기술 표준인 JPA의 Hibernate를 이용해 개발  
ORM이란 객체와 데이터베이스의 데이터를 자동으로 연결해 주는 것  
EntityManager는 JPA의 주요 기능으로, 객체를 영속성 컨테스트에 저장  
영속성 컨테스트는 1차 캐시의 기능과 트랜잭선을 지원함 
트랜잭션을 커밋할 경우 자동으로 EntityManager에 저장한 객체가 데이터베이스에 반영됨  

기존에 sqlite를 이용해 개발을 해보아서 SQL 구분에 대한 것은 대략적으로 알고 있었다.  
sqlite를 이용해 개발을 할 때, SQL 구문을 사용하는데 있어서 여러 불편한 부분이 있었고, 이를 해결할 수 있는 방법이 없을까 고민도 했었다.  
그러나 오늘 배운 ORM 기술을 사용한다면 SQL 구문을 중복해서 적지 않아도 되고, 각 SQL마다 실행 여부를 따져서 롤백을 하지 않아도 된다는 것이 매력적인 것 같다.  
물론 복잡한 기능 같은 경우는 SQL 구문을 사용하기도 하지만, 그럼에서 ORM 기술은 나에게 꽤 편맇람을 가져다주는 것 같다.  
다음번에 내가 개인 프로젝트를 개발한다면, ORM 기술을 사용해서 개발해보고 싶다.
