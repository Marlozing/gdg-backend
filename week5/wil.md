# 5주차 강의 내용
SOILD 원칙을 지키며 코드를 작성하는 방법  
Bean을 통해 객체를 관리하는 방법
 
SOILD 원칙을 지키며 프로젝트를 진행하기 위해 DI를 이용해 객체를 관리하는 방법을 배웠습니다.  
정해진 형식을 생성한 후 만약 내부 구조가 바뀌거나 여러 파일을 바꿔가며 사용하는 경우에도 DI를 이용해 객체를 관리하니 다른 부분을 변경하지 않고 오직 변경하는 부분만 변경하면
되니 유지보수 하기가 쉽고 예상치 못한 오류를 방지할 수 있겠다는 생각이 들었습니다. 
그동안의 수업을 통해 Bean에 대해 어렴풋이 알고 있었고, 어노테이션을 통해 자동으로 Bean이 등록된다는 것은 알고 있었으나, 이번 주차에서 수동으로도 Bean을 등록해보니 자동으로 
Bean이 등록되는 것이 얼마나 편리한 지 알 수 있었습니다.
